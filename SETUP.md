# Setup Instructions

## Что уже исправлено

- Починен AI-чат и улучшен системный промпт, чтобы ассистент отвечал как реальный бизнес-наставник.
- Исправлена ошибка конфигурации OpenAI: модель и ключ теперь разделены правильно.
- Усилен серверный пароль при регистрации, чтобы он совпадал с требованиями фронтенда.
- Подчищены переполнения текста и мобильная вёрстка, особенно в чате и на узких экранах.
- Убраны чувствительные значения из `.env`.

## Важно перед запуском

Архив, из которого собирался проект, содержал `node_modules` из другой ОС. Из-за этого Prisma и Rollup могли падать даже при корректном коде.

Поэтому после распаковки проекта обязательно установите зависимости заново на своей машине.

## 1. Настройте OpenAI

Откройте `backend/.env` и вставьте свой реальный ключ:

```env
OPENAI_API_KEY="sk-your-real-openai-key"
OPENAI_MODEL="gpt-4o-mini"
```

## 2. Установите зависимости заново

```bash
# backend
cd backend
npm install
npx prisma generate
npx prisma db push
npm run dev
```

```bash
# frontend
cd frontend
npm install
npm run dev
```

## 3. Откройте проект

- Frontend: `http://localhost:5173`
- Backend API: `http://localhost:4000/api`

## Что изменилось в AI

### AI Chat
- более сильный системный промпт под запуск и рост бизнеса
- упор на диагноз, приоритет, действия и метрики
- лучшее использование контекста Focus: задачи, участники, дедлайны, просрочки
- более понятные ошибки, если OpenAI ещё не настроен
- безопасный рендер ответов без `dangerouslySetInnerHTML`
- более удобные стартовые подсказки

### UI / mobile
- сообщения чата больше не вылезают за границы
- длинные строки и тексты лучше переносятся на iPhone и других телефонах
- шапка, табы, chips и служебный текст ведут себя аккуратнее на узких экранах
- уменьшено ощущение «растянутого» текста в некоторых блоках
